# kupecloud/scoop-bucket

Official [Scoop](https://scoop.sh) bucket for [Kupe Cloud](https://kupe.cloud) — managed Kubernetes clusters in seconds.

## Install

```powershell
scoop bucket add kupe https://github.com/kupecloud/scoop-bucket
scoop install kupe
```

Updates land via `scoop update kupe`.

## What's in this bucket

| App | Description | Upstream |
|-----|-------------|----------|
| `kupe` | Official CLI for Kupe Cloud | [kupecloud/kupe-cli](https://github.com/kupecloud/kupe-cli) |

Manifests in this repo are generated by [goreleaser](https://goreleaser.com) on each release of the upstream tool. Please don't edit them by hand — your change will be overwritten on the next release.

## Issues and contributions

Please report issues, request features, or contribute to the CLI itself at [kupecloud/kupe-cli](https://github.com/kupecloud/kupe-cli/issues). This bucket holds package metadata only and is auto-generated.

For bucket-specific issues (the `scoop install` step itself failing, manifest schema errors), open an issue here.

## Verifying release artifacts

Each release's `checksums.txt` is signed with [Cosign](https://github.com/sigstore/cosign) keyless signing via GitHub Actions. To verify on Windows:

```powershell
$TAG = "v0.1.0"   # replace with the release you installed
gh release download $TAG -R kupecloud/kupe-cli -p '*checksums.txt*'
cosign verify-blob `
  --certificate "kupe_$($TAG.TrimStart('v'))_checksums.txt.pem" `
  --certificate-identity-regexp 'https://github.com/kupecloud/kupe-cli/.+' `
  --certificate-oidc-issuer https://token.actions.githubusercontent.com `
  --signature "kupe_$($TAG.TrimStart('v'))_checksums.txt.sig" `
  "kupe_$($TAG.TrimStart('v'))_checksums.txt"
```

> **Note:** the `.pem` signing certificate is only attached to releases newer than v1.8.1. Earlier releases do not ship the certificate asset and cannot be keyless-verified with this procedure.
